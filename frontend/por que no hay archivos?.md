# Frontend — Justificación

El sistema fue desarrollado utilizando PHP bajo un enfoque monolítico, donde la lógica del servidor y la interfaz de usuario se encuentran integradas en los mismos archivos (`login.php`, `registro.php`, `dashboard.php`).

Aunque no existe una separación tradicional de frontend, el sistema sí incluye elementos de interfaz como formularios, estilos CSS y validaciones con JavaScript, los cuales están embebidos directamente en los archivos PHP.

Esta decisión se tomó para simplificar el desarrollo en una etapa inicial, reduciendo la complejidad del sistema. Como mejora futura, se contempla una posible separación del frontend en una arquitectura independiente basada en APIs.
